# 一、浮动

# 1.1传统网页布局的三种方式

网页布局的本质——用CSS来摆放盒子，把盒子摆放到相应位置。

三种：

  • 标准流（普通流/文档流）

    就是按标签规定好的默认方式排列。1、块级独占一行，从上向下排列。2、行内元素。3、行内块元素。

    标准流是最基本布局网页的方式。

  • 浮动

  • 定位

在实际开发中，一个页面基本包含了这三种布局方式。（后面移动端学习新的布局方式 ）

注：课程里提到table是属于块级元素，但在代码里是table类型，所以暂时不想太多

# 1.2浮动

有很多布局效果，标准流没有办法完成，此时就可以利用浮动完成布局，因为浮动可以改变元素标签默认的排列方式。

最典型应用：让多个块级元素一行内排列显示。

！多个块级元素纵向排列找标准流，横向排列找浮动。

float属性用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘。（浮动贴着浮动连在一起）

选择器{float;属性值;}

none:不浮动（默认）

left:左浮

right:右浮

# 1.3浮动特性

1、浮动元素会脱离标准流（脱标）移动到指定位置。

浮动的盒子不再保留原先的位置。//可能会产生浮动的盒子压住标准流的盒子。（浮动相当于更上层了）

2、浮动的元素会一行内显示 并且元素顶部对齐。//不过装不下时会自动换行。

3、浮动的元素会具有行内块元素的特性。//无论是行还是块元素。

浮动元素经常和标准流父级搭配使用：

先用标准流的父元素排列上下位置，之后内部子元素采取浮动排列左右位置，符合网页布局的第一准则。

# 1.6清除浮动

由于父级盒子很多情况下，不方便给高度，但是子盒子浮动又不占有位置，最后父级盒子高度为0时，就会影响下面的标准流盒子。

# 二、定位

以下情况使用标准流和浮动不能实现：

1、某个元素可以自由的在一个盒子内移动位置，并且压住其他盒子。

2、当滚动窗口时，有些盒子固定在屏幕的某个位置。

需要用定位来实现，所以：

1、浮动可以让多个块级盒子一行内没有缝隙显示，经常用于横向排列盒子。

2、定位则是可以让盒子自由在某个盒子内移动位置或固定在屏幕中某个位置，并且可以压住其他盒子。

# 2.1定位组成

定位：将盒子定在某一个位置，所以定位也是在摆放盒子，按照定位的方式移动盒子。

定位=定位模式（定位方式）+边偏移（最终位置）

定位模式

position: static静态定位/relative相对/absolute绝对（在父盒子移动，压住其它盒子）/fixed固定定位（固定在屏幕某个地方不动）。

边偏移

（这些属性只存在于定位中）

属性 值 描述
top px 顶端偏移量，定义元素相对其父元素上边线的距离
bottom px 底部
left px 左侧
right px 右侧

# 2.2静态定位 static

默认定位方式，无定位的意思;

选择器{position:static;}
1

按照标准流特性摆放位置，没有边偏移。

在布局中很少用到

# 2.3相对定位 relative(重要)

元素在移动位置时，是相对它原来的位置来说的。

如top:100px，是根据原来坐标向下移动100px.

原来在标准流的位置继续占有，后面的盒子仍然以标准流的方式对待它。（不脱标，继续保留原来位置）

它最典型应用是给绝对定位当爹的。

# 2.4绝对定位absolute(重要)

绝对定位是元素在移动位置的时候，是相对它祖先元素来说的。

特点：

1、如果没有祖先元素或者祖先元素没有定位（标准流或浮动），则以浏览器为准定位（Document文档）

2、如果祖先元素有定位（相对、绝对、固定），则以最近一级的有定位祖先元素为参考点移动位置。

3、绝对定位不再占有原来的位置（比浮动飘的还高）。（脱标）

# 2.5子绝父相的由来

孩子绝对定位，父亲相对定位。

1、子级绝对定位，不会占有位置，可以放到父盒子的任何一个地方，不会影响其它的兄弟盒子。

2、父盒子需要加定位限制子盒子在父盒子内显示。

3、父盒子布局时，需要占有位置，因此父亲只能是相对定位。

子绝父相不是永远不变的，根据需要来变化。

# 2.6 固定定位fixed(重要)

元素固定于浏览器可视区的位置。主要作用场景：可以在浏览器页面滚动时元素的位置不会改变。

特点：

1、以浏览器的可视窗口为参照点移动

跟父元素没有任何关系。不随滚动而滚动。

2、固定定位不占有原来的位置。（脱标，可以看作是一种特殊的绝对定位）

固定定位小技巧：固定在版心右侧位置:

left: 50%；margin-left: 版心宽度的一半。

# 2.7粘性定位sticky

可以认为是相对定位和固定定位的混合。

特点：

1、以浏览器的可视窗口为参照点移动元素（固定定位）

2、粘性定位占有原先的位置（相对定位特点）

3、必须添加top、left、right、bottom其中的一个才有效。（不加的话，就会被当成相对定位了）。如top：0px，是指当盒子移动列距离顶部0px时就变成固定定位效果了。

注：不常用，兼容性较差，ie不支持。

# 2.8定位叠放次序z-index

使用定位布局时，可能会出现盒子重叠的情况。此时，可以用不用z-index来控制盒子的前后次序（z轴）

选择器{z-index: 1;}
1
  • 数值可以是正、负整数或0，默认是auto,数值越大，盒子越靠上。（1比auto级别高）
  • 如果值相同，按书写顺序，后来居上。
  • 数字不能加单位
  • 只有定位才有这个属性

# 2.9定位的拓展

绝对定位的盒子居中：

加了绝对定位的盒子不能通过margin: 0 auto水平居中，但可以通过下面方法：

left: 50%,margin-left:- 盒子的一半。

定位特殊特性：

绝对定位或固定定位也和浮动类似（相当于也是变成行内块了）

1、行内元素添加绝对或固定定位，可以直接设置高度和宽度。

2、块级元素添加绝对或固定定位，如果不给宽高，默认大小是内容的大小。

脱标的盒子不会触发外边距塌陷（也不浮动差不多）

绝对定位（固定定位）会完全压住盒子

浮动元素不同，只会压住它下面标准流的盒子，但是不会压住下面标准流盒子里的文字、图片。（因为浮动最初被设计出来就是做文字环绕效果的。）

绝对定位（固定定位）会完全压住盒子，包括其内容。

如果同时定义left:0 和right:0 因为是不同的属性所以不会冲突，但会秉持先左后右，先上再下的原则优先显示左和上。

设置圆角矩形时，半径尽量不写小数点，取整就行。

# 2.10总结

1、垂直的块级盒子显示用标准流布局

2、多个块级盒子水平显示用浮动布局

3、如果元素自由在某个盒子内移动就用定位布局。

# 2.11元素的显示与隐藏

本质：让一个元素在页面中隐藏或者显示出来。

1、display显示隐藏！！！

：none元素的隐藏

: block：除了转化为块级元素外，同时还有显示元素的意思

display隐藏元素后，不再占有原来的位置。

2、visibility显示隐藏

:visible

:hidden

隐藏后，继续占有原来的位置,如果它本身本来就占有位置，隐藏后，依旧不占有位置。

3、overflow溢出显示隐藏

: visible(默认)

:hidden(多出来的部分隐藏)

:scroll（不管有没有溢出都显示滚动条）

:auto(在需要的时候添加滚动条)

如果有定位的盒子，而且有超出的部分，慎用overflow:hidden,因为会隐藏多余部分。

# 二、CSS高级

# 2.1精灵图

一个图片往往会应用很多很小的背景作为修改，图像过多，服务器过频繁接收和发送请求图片，造成服务器压力过大，降低页面的加载速度。

因此，使用CSS精灵技术（CSS Sprites/CSS雪碧)可以有效地减少服务器接收和发送请求的次数。

原理：将很多小图片合成到一张大的图片中。

使用精灵图核心：

1、主要用于背景图片使用。把多个小背景图片整合到一张大图中。而非背景图片因为需要经常更换，所以不用精灵图。

2、移动背景图片位置，使用background-position。//测量精灵图的坐标可以用FW.在b站P252.

3、移动距离就是这个目标图片的x和y坐标。注意网页的坐标有所不同。向上向左是负值。

# 2.2字体图标

常用于网页上一些通用的小图标。

精灵图的缺点：

1、图片较大

2、放大缩小会失真

3、图片制作后更换复杂。

字体图标iconfont。展示的是图标，本质属于字体。

优点：

1、轻量级：一个图标字体比一系列图像要小，一旦字体加载了，图标就会马上渲染出来 ，减少了服务器请求。

2、灵活性。本质其实是文字，可以很随意地改变颜色、产生阴影、透明效果、旋转等

3、兼容性：几乎支持所有的浏览器。

总结：

1、结构样式简单的小图标，用字体图标。

2、结构样式复杂的小图片，用精灵图。

# 2.5 vertical-align 属性应用

CSS 的 vertical-align 属性使用场景：

经常用于设置图片或者表单(行内块元素）和文字垂直对齐。

用于设置一个元素的垂直对齐方式（在垂直方向上的对齐方式，也可以说是两个元素的水平对齐），但是它只针对于行内元素或者行内块元素有效。

vertical-align : baseline（默认） | top | middle | bottom
1

图片、表单都属于行内块元素，默认的 vertical-align 是基线对齐。

此时可以给图片、表单这些行内块元素的 vertical-align 属性设置为 middle 就可以让文字和图片垂直居中对齐了。

一般都是让行内块元素来弄垂直居中。

bug：图片底侧会有一个空白缝隙，原因是行内块元素会和文字的基线对齐。而边框需要照顾到他附近可能有的文字，所以会有点小缝隙。

主要解决方法有两种：

  1. 给图片添加 vertical-align:middle | top| bottom 等。 （提倡使用的）

  2. 把图片转换为块级元素 display: block;
